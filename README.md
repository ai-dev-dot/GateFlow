# 闸机 GateFlow

让企业AI，安全有序地流动

闸机 GateFlow 是企业内部所有大模型调用的统一入口。
它帮你管住数据泄露风险，控住AI使用成本，理清所有使用行为，
让全公司从"个人偷偷用AI"，变成"全公司放心用AI"。

## 企业AI落地的5大灾难

这是所有企业现在正在经历的真实痛点，也是闸机诞生的原因：

1.  **数据泄露爆炸**：员工把客户名单、合同、代码、财务数据直接粘贴到公网大模型，核心资产随时可能泄露且完全不可追溯
2.  **成本完全失控**：各部门各自买账号、各自充钱，一年花几百万不知道谁用了、用在哪了、产生了什么价值
3.  **管理一片混乱**：没有统一标准，有人用GPT-4o，有人用DeepSeek，有人用本地模型；不同部门知识库不互通，重复建设
4.  **合规审计空白**：金融、医疗、政务等强监管行业，AI生成内容无法审计、无法溯源，出问题企业和管理者承担法律责任
5.  **集成效率极低**：每个业务系统都要单独对接不同大模型API，重复开发，维护成本极高；模型切换时所有系统都要改代码

闸机 GateFlow，一次性解决以上所有问题。

> 技术团队在 Claude Code、Codex、Dify、Coze、LangChain、Cursor 等工具里，把 API 地址指向闸机，就能让 Agent 走企业统一通道。一行配置，零代码改动。


## 核心功能

按企业落地优先级排序：

### 统一模型API网关（MVP核心）
- 一套标准RESTful API，统一封装OpenAI、Anthropic、DeepSeek、字节豆包、百度文心等所有主流大模型
- 企业级API Key池管理，所有员工共享使用，无需个人申请账号
- 智能流量调度与故障自动降级，某个模型宕机时无缝切换到备用模型
- 细粒度速率限制，按用户、角色、部门设置不同的请求频率

### 企业级权限与身份管理（MVP核心）
- 三层RBAC权限控制：模型级别、功能级别、数据级别
- 原生支持企业微信、钉钉、飞书、LDAP、AD单点登录，员工用现有账号直接登录
- 临时授权机制，可给外部顾问、外包人员开通限时、限功能的访问权限
- 支持部门隔离，不同部门的数据和知识库完全独立

### 数据安全与审计溯源（MVP核心）
- 全量永久日志记录，保存所有请求和响应的完整内容，留存时间可配置
- 内置敏感数据检测与自动脱敏，支持自定义敏感词库和正则表达式
- 强大的审计检索能力，支持按用户、部门、时间、关键词全文检索
- 异常行为自动检测与告警，识别高风险使用行为

### 成本分析与管控（MVP推荐）
- 多维度成本统计：按部门、用户、模型、功能维度实时统计用量和费用
- 预算管理与自动告警，给每个部门、每个用户设置月度/季度预算阈值
- 智能成本优化建议，分析哪些请求可以用更便宜的模型处理
- 自动生成部门分摊账单，支持导出Excel

### 私有知识库与工具集成（后续迭代）
- 统一RAG引擎，企业只需上传一次文档，所有员工和应用都可使用
- 自动同步Confluence、SharePoint、Notion、企业网盘、数据库等内部数据源
- 统一工具调用框架，支持网页搜索、代码执行、SQL查询、自定义API调用

### 影子AI治理（独家特色）
- 预留DLP系统集成接口，与企业现有数据防泄漏系统无缝对接
- 提供企业浏览器管控方案，自动重定向未授权AI网站到闸机
- 个人API Key检测与拦截，引导员工使用企业统一账号
- 完整的影子AI治理方法论，帮助企业建立"疏堵结合"的管理体系

## 为什么选择闸机 GateFlow

| 产品类型 | 代表产品 | 核心劣势 | 闸机 GateFlow 优势 |
|---------|---------|---------|-------------------|
| 大模型厂商企业版 | OpenAI企业版、DeepSeek企业版 | 只支持自家模型；数据发给厂商；价格昂贵 | 支持所有模型；完全私有化部署，数据不出内网；永久免费 |
| 商业AI网关 | 各类SaaS网关产品 | 价格昂贵；定制化能力差；数据经过第三方 | 开源免费；100%自主可控；可根据企业需求任意二次开发 |
| 开源RAG平台 | LangChain、LlamaIndex | 只做RAG，没有网关、权限、审计、成本功能 | 完整的企业AI基础设施，RAG只是其中一个模块 |
| 自研系统 | 企业内部开发 | 开发周期长；维护成本高；踩坑无数 | 开箱即用；持续迭代；社区支持；踩过所有你会踩的坑 |

## 快速开始

### 一键部署（推荐）
```bash
# 克隆仓库
git clone https://github.com/your-username/gateflow.git
cd gateflow

# 创建并激活虚拟环境
python -m venv venv
# Windows
venv\Scripts\activate
# Mac/Linux
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 复制环境变量文件
cp .env.example .env

# 编辑环境变量，配置数据库和大模型API Key
# vim .env

# 初始化数据库
python main.py init-db

# 启动服务
python main.py run
```

访问 `http://localhost:3000` 即可使用，默认管理员账号：`admin@gateflow.local`，密码：`admin123`

### Docker部署
```bash
docker-compose up -d
```
